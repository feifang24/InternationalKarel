গুরুত্বপূর্ণ বাক্য পিকিং

1. শর্তাধীন বিবৃতি প্রবর্তন

Pothole ভর্তি আগে Potholeপূরণকরুন() ফাংশন, কারেল হয়তো অন্য কোন মেরামতের ক্রু ইতিমধ্যে একটি কোণার সঙ্গে কোণার গর্ত ভরা কিনা চেক করতে চান। যদি তাই হয়, কারেল দ্বিতীয় একটি ডাউন করা প্রয়োজন হয় না। প্রোগ্রামের প্রেক্ষাপটে এই ধরনের চেকগুলি উপস্থাপন করার জন্য আপনাকে যদি বিবৃতিটি ব্যবহার করতে হয় তবে নিম্নলিখিত দুটি ফর্মগুলির মধ্যে একটিটি ব্যবহার করতে হবে: 


if (শর্তাধীনটেস্ট()) {
   বিবৃতিশুধুমাত্রসত্যহলেইকার্যকরকরাহবে();
}
অথবা 

if (শর্তাধীনটেস্ট()) {
   শর্তটিকার্যকরহলেবিবৃতিটিকার্যকরকরাহবে();
} else {
   অবস্থামিথ্যাহলেবিবৃতিকরাহবে();
}
যদি বিবৃতির প্রথম ফর্মটি শুধুমাত্র নির্দিষ্ট শর্তগুলির অধীনে কোনও কাজ সম্পাদন করতে চান তবে এটি কার্যকর। দ্বিতীয়টি কার্যকর হলে দুটি বিকল্প কোর্স বেছে নিতে হবে।

2. পুনরাবৃত্তিমূলক বিবৃতি প্রবর্তন

যেহেতু আপনি এই উদাহরণ থেকে জানেন যে পূরণ করার জন্য ঠিক পাঁচটি গর্ত রয়েছে, আপনার যে নিয়ন্ত্রণ বিবৃতিটি প্রয়োজন তা হল লুপের জন্য, যা নির্দিষ্ট করে যে আপনি কোন ক্রিয়াকলাপটিকে পূর্বনির্ধারিত বার পুনরাবৃত্তি করতে চান। লুপ সিনট্যাক্স জন্য এই মত দেখায়: 


for (আমি = 0; আমি < পুনরাবৃত্তিসংখ্যা; আমি++) {
   বিবৃতিপুনরাবৃত্তিকরা();
}
উদাহরণস্বরূপ, আপনি যদি পরিবর্তন করতে চান দুইPotholesপূরণকরুন.js প্রোগ্রামটি যাতে পাঁচটি সমান স্পেসেড গর্ত পূরণ করার জটিল সমস্যাটি সমাধান করে, আপনাকে যা করতে হবে তা নিম্নোক্ত কোডটি লিখুন: 

function পাঁচPotholesপূরণকরুন() {
   for (আমি = 0; আমি < 5; আমি++) {
      পদক্ষেপ();
      Potholeপূরণকরুন();
      পদক্ষেপ();
   }
}
পুনরাবৃত্তি বিবৃতিটি শুধুমাত্র তখনই উপকারী হবে যখন আপনি অগ্রগতির সংখ্যাটি সম্পাদন করার জন্য অগ্রিমভাবে জানেন।

3. বিচ্ছেদ

যখনই আপনি কোনও প্রোগ্রামিং সমস্যা সমাধান করতে শুরু করেন-কারও কারেলের প্রোগ্রামটি বা কোনও উন্নত প্রোগ্রামিং ভাষাতে লেখা আছে কিনা-আপনার প্রথম কাজটি কীভাবে সম্পূর্ণ সমস্যাটি উপপ্রয়োগ নামে পরিচিত ক্ষুদ্র টুকরাগুলিতে বিভক্ত করা হয় তা বোঝার জন্য প্রতিটিটি বাস্তবায়িত করা যেতে পারে একটি পৃথক ফাংশন হিসাবে। যে প্রক্রিয়া decomposition বলা হয়।

4. decomp কংক্রিট উদাহরণ

যদিও আপনি চারটি পূর্বনির্ধারিত নির্দেশাবলী ব্যবহার করে এই সমস্যার সমাধান করতে পারেন তবে আপনি আপনার প্রোগ্রামের কাঠামোর উন্নতি করতে ফাংশনগুলি ব্যবহার করতে পারেন। অন্য কিছু না থাকলে, আপনি প্রোগ্রামটিকে ছোট করতে এবং তার অভিপ্রায়টি আরও পরিষ্কার করতে টারআউন্ডআউন্ড এবং টার্নিট ব্যবহার করতে পারেন। আরো গুরুত্বপূর্ণভাবে, আপনি সমস্যাটি উপ-প্রবলেমগুলিতে সমস্যাটি ভেঙ্গে ফেলতে এবং তারপর সেই সমস্যাগুলিকে স্বাধীনভাবে সমাধান করতে ব্যবহার করতে পারেন। উদাহরণস্বরূপ, আপনি নিম্নোক্ত উপপাদ্যগুলিতে পোথোল পূরণ করার সমস্যাটি ভাগ করতে পারেন:
1. দ্বিতীয় এভিনিউতে প্রথম পথোল পৌঁছানোর জন্য এক ব্লক এগিয়ে যান।
2. এটি একটি beeper ড্রপ দ্বারা pothole পূরণ করুন।
3. 5 তম এভিনিউতে দ্বিতীয় পোথোল পৌঁছানোর জন্য তিনটি ব্লক এগিয়ে যান।
4. এটি একটি beeper ড্রপ দ্বারা pothole পূরণ করুন।
5. পছন্দসই চূড়ান্ত অবস্থান পৌঁছানোর জন্য একটি ব্লক এগিয়ে যান।
আপনি যদি এভাবে সমস্যার বিষয়ে চিন্তা করেন, তবে প্রোগ্রামটি আপনার সমস্যার গঠন সম্পর্কে আপনার ধারণাটি প্রতিফলিত করার জন্য ফাংশনগুলি ব্যবহার করতে পারে

5. শীর্ষ-ডাউন নকশা

ধাপে ধাপে পরিমার্জনা কেন্দ্রীয় ধারণাটি হল যে আপনি উপরের থেকে আপনার প্রোগ্রামের নকশা শুরু করতে হবে, যা ধারণাটির সর্বোচ্চ স্তর এবং সর্বাগ্রে প্রোগ্রামের স্তরটিকে বোঝায়। এই স্তরে, বিপার টাওয়ার সমস্যাটি স্পষ্টভাবে তিনটি স্বাধীন পর্যায়ে বিভক্ত। প্রথম, কারেল সব beepers সংগ্রহ করতে হবে। দ্বিতীয়ত, কারেলকে তাদের শেষ ছদ্মবেশে জমা দিতে হবে। তৃতীয়, কারেল তার বাড়ির অবস্থান ফিরে করতে হবে। এই রূপরেখাটি সমস্যাটির নিম্নলিখিত বিচ্ছেদকে প্রস্তাব করে: 


function বিপারটাওয়ারসংগ্রহকরুন() {
   সবBeepersসংগ্রহকরুন();
   সবBeepersড্রপ();
   বাড়িতেফিরে();
}
এই স্তরে, সমস্যা বুঝতে সহজ। যদিও আপনি CollectBeeperTowers এর শরীরের ক্রিয়াকলাপগুলির জন্য কোডটি লেখেন নি, তবুও নিজেকে বিশ্বাস করা গুরুত্বপূর্ণ যে, যতক্ষণ আপনি বিশ্বাস করেন যে আপনি যেসব ফাংশন লিখতে যাচ্ছেন সেগুলি উপপ্রয়োগগুলিকে সঠিকভাবে সমাধান করবে, আপনার কাছে এটি সমাধান হবে সম্পূর্ণরূপে সমস্যা।

6. কারেল পরিভাষা

কারেলের পৃথিবীটি পূর্ব থেকে পূর্ব দিকে এবং রাস্তায় দক্ষিণ থেকে উত্তর দিকে চলমান রাস্তায় সংজ্ঞায়িত। রাস্তা এবং একটি এভিনিউ ছেদ একটি কোণার বলা হয়। কারেলকে শুধুমাত্র কোণায় স্থাপন করা যেতে পারে এবং চারটি মানক কম্পাস নির্দেশাবলীর (উত্তর, পূর্ব, দক্ষিণ, এবং পশ্চিমে) সম্মুখীন হওয়া আবশ্যক। নিচের নমুনা জগতে, কারেল 1 ম স্ট্রিট এবং 1 ম অ্যাভিনিউ কোণে পূর্ব দিকে মুখোমুখি হচ্ছে: কারেলের জগতের অন্যান্য উপাদান এই উদাহরণে দেখা যেতে পারে। কারেলের সামনে ধূসর, হীরা আকৃতির বস্তুটি একটি বীপার। রিচ প্যাটিসগুলি বীপারকে বর্ণনা করে "প্লাস্টিকের কোণগুলি যা একটি শান্ত বীপ শব্দটি নির্গত করে।" এই শব্দগুলি শুধুমাত্র শ্রবণযোগ্য হলে কারেল এবং বিপার একই কোণে থাকে। উদাহরণস্বরূপ, কারেল চিত্রের মত বীপার থেকে অবগত নন এবং কারেলের পরবর্তী কোণে অগ্রসর হলে কেবল বীপ আবিষ্কার করবে। ডায়াগ্রাম দৃঢ় লাইন দেয়াল হয়। কারেলের পৃথিবী সবসময় প্রান্ত বরাবর দেয়ালে রয়েছে এবং অভ্যন্তরীণ দেয়ালও থাকতে পারে।

7. ফাংশন সংজ্ঞায়িত

কারেল প্রোগ্রামিং ভাষার সবচেয়ে শক্তিশালী বৈশিষ্ট্যগুলির মধ্যে একটি হল নতুন ফাংশন সংজ্ঞায়িত করার ক্ষমতা। যখনই আপনার কারেল অপারেশনগুলির একটি ক্রম থাকে যা কিছু কার্যকর কার্য সম্পাদন করে - যেমন ডান দিকে বাঁক-আপনি সেই ক্রমটি একটি নাম দিতে পারেন। একটি নতুন নাম অধীনে নির্দেশাবলীর একটি ক্রম encapsulating অপারেশন একটি ফাংশন সংজ্ঞায়িত বলা হয়। একটি ফাংশন সংজ্ঞায়িত করার জন্য বিন্যাস এই মত দেখাচ্ছে: 


function নাম() {
   শরীরেরবিবৃতি();
}
এই প্যাটার্নের প্রথম শব্দ হল ফাংশন সংজ্ঞায়িত করার জন্য কারেল এবং জাভাস্ক্রিপ্টে ব্যবহৃত হয়। একটি প্রোগ্রামিং ভাষাতে নির্দিষ্ট, পূর্বনির্ধারিত অর্থ রয়েছে এমন শব্দগুলিকে কীওয়ার্ড বলা হয়।
